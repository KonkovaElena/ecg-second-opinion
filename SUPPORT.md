# Support

## What This Repository Supports Today

This repository supports public discussion and contribution around the current ECG Second Opinion engineering baseline, workflow behavior, and public GitHub trust surfaces.

Use support channels conservatively:

1. reproducible bugs or runtime regressions: GitHub Issues with the provided templates;
2. documentation drift or scope confusion: GitHub Issues with links to the affected sections;
3. proposed code changes: Pull Requests with updated docs and validation notes;
4. publication-readiness or trust-surface questions: review `README.md`, `CONTRIBUTING.md`, and `SECURITY.md` first.

## What Is Not A Support Channel

1. private security reports in public issues;
2. secrets, credentials, regulated data, or patient-identifying information;
3. emergency operational support or clinical incident handling.

## Security

Security reports must follow the process in `SECURITY.md`.

## Response Expectations

This is a research and engineering repository, not a staffed product support desk.

Best-effort triage is the intended operating model until a dedicated maintainer team is declared.
