## Strategic Next Sprints

### Sprint 1 — Durable Infrastructure

Goal: remove the highest operational risks before any serious staging or external trial.

Tasks:

1. replace `setImmediate()` dispatch in `src/app.ts` with a durable queue or outbox-backed dispatch model
2. add bounded retry/backoff and a dead-letter or replayable failure path
3. replace the in-memory repository with durable persistence, targeting PostgreSQL for real environments
4. preserve audit-trail integrity, safety-flag semantics, and aggregate reconstitution under persistence
5. add tests that prove a case cannot remain silently stranded in `InferencePending`

Acceptance criteria:

- inference dispatch is durable and observable
- restart or process crash does not lose case state
- retry policy is explicit and bounded
- failure handling remains auditable

### Sprint 2 — Regulatory and Lifecycle Compliance

Goal: correct external positioning and prepare a credible future clinical path.

Tasks:

1. keep all operator-facing materials framed as research use only until a formal device strategy exists
2. document a SaMD-oriented U.S. pathway analysis: `510(k)` versus `De Novo` depending on intended claims and predicates
3. define a PCCP-style change-governance strategy for future AI/ML model updates
4. create GMLP-oriented work products, including post-market or post-deployment drift monitoring requirements
5. prepare EU AI Act evidence surfaces for medical deployment, especially human oversight and data governance

Acceptance criteria:

- no active doc claims a non-device CDS exemption for a future ECG-signal-analysis deployment
- U.S. and EU compliance workstreams are explicit
- model-update governance is documented before real inference rollout

### Sprint 3 — Clinical Data Interoperability

Goal: make the project interoperable with external clinical ecosystems rather than only structurally FHIR-like.

Tasks:

1. add SNOMED CT / LOINC-coded vocabularies to FHIR export
2. add `meta.profile` support and stronger identifier-system conventions
3. define an enterprise ECG data layer for inbound/outbound **HL7 aECG** and **DICOM Waveform** support
4. add validator-backed FHIR tests and at least one standards-mapping test path

Acceptance criteria:

- FHIR remains human-readable and machine-usable
- coded values are explicit
- enterprise waveform standards have a concrete mapping plan

### Sprint 4 — Real Inference Stack

Goal: replace the metadata fallback stub with a research-backed inference subsystem.

Tasks:

1. implement a Python 1D-CNN worker trained on PTB-XL or an equivalent research dataset
2. add MC-Dropout uncertainty estimation and retain the existing safety-policy hooks
3. add Grad-CAM or equivalent explainability artifacts to the inference contract
4. define drift monitoring around performance, concept, and data shifts once a real model is deployed

Acceptance criteria:

- the TypeScript control plane remains decoupled from model execution
- uncertainty and explainability are first-class outputs
- deployment monitoring extends beyond static pre-release validation

## Recommended Execution Order

1. Sprint 1 first: it addresses the most serious runtime and data-loss risks.
2. Sprint 2 second: it corrects external positioning and prepares a compliant path before stronger clinical claims.
3. Sprint 3 third: it improves clinical integration once runtime and storage are credible.
4. Sprint 4 fourth: it adds the real model stack on top of a durable, better-governed platform.

## Verification Rails For Future Sprints
